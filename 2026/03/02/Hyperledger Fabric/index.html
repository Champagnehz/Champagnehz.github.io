溯源区块链 (Hyperledger Fabric)

基于 Hyperledger Fabric 的产品溯源区块链，支持数据上链、流动路径追踪、数据校验。
源码：https://github.com/Champagnehz/fabric-traceability

2026-02-27 搭建完成

部署状态

  • Fabric 网络运行中（Orderer + 2 Peer + 3 CA）
  • 链码 traceability v1.0 已部署到 tracechannel
  • REST API 网关运行在 http://localhost:3000
  • Windows 通过 WSL2 自动端口转发直接访问

注意事项

  • Docker 镜像加速已配置（/etc/docker/daemon.json）
  • Go 代理使用 goproxy.cn
  • npm 使用 npmmirror.com
  • API 中 evaluateTransaction 返回 Uint8Array，需用 Buffer.from(result).toString('utf8') 解码

数据流转过程（以”创建产品”为例）

数据校验原理

每次写入数据时，链码会：

  1. 将产品信息（不含 dataHash 字段）序列化为 JSON
  2. 计算 SHA-256 哈希值，存入 dataHash 字段
  3. 校验时重新计算哈希，与链上存储的哈希比对
  4. 如果一致说明数据未被篡改，不一致则数据存在问题

文件结构

主要功能：

CreateProduct → 数据上链 + 生成 SHA-256 指纹
AddTrace → 追加流转记录
QueryProduct → 查询溯源信息
VerifyData → 哈希校验防篡改
GetHistory → 查询区块链历史版本

API 接口

方法 路径 功能 Body 参数
GET / API 说明页 -
POST /api/product 数据上链（创建产品） productID, name, origin, owner
POST /api/product/:id/trace 增加流动路径 operator, location, action, remark
GET /api/product/:id 溯源查询 -
GET /api/product/:id/verify 数据校验（SHA-256 比对） -
GET /api/product/:id/history 区块链交易历史 -
GET /api/health 健康检查 -

链码功能

  • CreateProduct: 产品信息上链，自动生成创建时间和数据哈希
  • AddTrace: 向产品流转链追加记录（生产→运输→入库→质检→销售）
  • QueryProduct: 返回产品完整信息及全部流转路径
  • VerifyData: SHA-256 哈希重算比对，验证链上数据是否被篡改
  • GetHistory: 利用 Fabric 账本不可变特性查询所有历史版本
